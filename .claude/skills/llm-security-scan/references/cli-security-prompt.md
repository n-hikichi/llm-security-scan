# セキュリティレビュープロンプト

あなたはセキュリティ専門家として、Vue.js フロントエンドクライアントアプリケーションの包括的なセキュリティレビューを実施してください。

## レビュー対象
- プロジェクト: heresme-cli (Here's Me Client Application)
- 技術スタック: Vue.js 2.5.16, Vuex 3.0.1, Axios 0.24.0, Vuetify 1.4.2
- 言語: JavaScript (ES6)

## レビュー重点領域

### 1. 認証・認可の実装（クライアント側）
以下の観点で詳細にレビュー:
- 認証トークン（JWT）のクライアント側での保存方法（localStorage、sessionStorage、Cookie）
- トークンの有効期限チェックと自動更新の実装
- 認証状態の管理（Vuex store）とセキュアな実装
- ログイン・ログアウト処理の安全性
- パスワード入力のクライアント側検証とマスキング
- 認証エラー時の適切なハンドリング
- 自動ログアウト機能の実装

### 2. API 通信のセキュリティ
以下の観点で詳細にレビュー:
- API エンドポイント URL のハードコーディングと機密情報の露出
- Axios インターセプターでのセキュリティヘッダー設定
- HTTPS の強制使用
- API レスポンスの検証とサニタイゼーション
- 認証トークンの送信方法（Header、Cookie など）
- API エラー時の情報漏洩リスク
- CORS 設定との整合性

### 3. XSS（クロスサイトスクリプティング）対策
以下の観点で詳細にレビュー:
- ユーザー入力の表示時のエスケープ処理
- v-html ディレクティブの使用箇所とサニタイゼーション
- DOM ベースの XSS 脆弱性
- 外部リソース（CDN、画像、スクリプト）の信頼性
- Content Security Policy (CSP) の設定
- innerHTML や outerHTML の直接操作
- 動的コンポーネント生成時のリスク

### 4. 依存関係の脆弱性
以下の観点で詳細にレビュー:
- 既知の脆弱性を持つパッケージの特定
- パッケージバージョンの古さの評価（Vue.js 2.5.16 など）
- 直接依存と間接依存の脆弱性
- 推奨される更新パスと互換性の問題
- 依存関係の脆弱性チェック結果

## 追加の重要なセキュリティ観点

### クライアント側データ保護
- ブラウザストレージ（localStorage、sessionStorage）への機密情報の保存
- メモリ内での機密データの扱い
- ブラウザキャッシュによる情報漏洩リスク
- クライアント側でのログ出力による情報漏洩

### エラーハンドリングと情報漏洩
- エラーメッセージでの詳細情報の漏洩
- コンソールログでの機密情報の記録
- デバッグ情報の本番環境での無効化
- 開発者ツールでの情報露出

### ルーティングとナビゲーションのセキュリティ
- Vue Router のナビゲーションガードによる認証チェック
- 未認証時のリダイレクト処理
- URL パラメータでの機密情報の送信
- Open Redirect 脆弱性

### 入力検証とサニタイゼーション
- フォーム入力のクライアント側検証
- ファイルアップロード時の検証（タイプ、サイズ、拡張子）
- URL や画像パスの検証
- ユーザー入力の適切なエスケープ処理

### サードパーティライブラリのセキュリティ
- OpenLayers、Vuetify などの外部ライブラリの脆弱性
- CDN 経由のリソース読み込みのリスク
- Subresource Integrity (SRI) の使用

## レビュー出力形式

各脆弱性について、以下の形式で報告してください：

### 脆弱性タイトル
**重要度**: Critical / High / Medium / Low

**場所**: ファイルパス:行番号

**説明**:
脆弱性の詳細な説明

**影響**:
この脆弱性による潜在的な影響やリスク

**再現手順**:
可能であれば、脆弱性を再現する手順

**修正提案**:
具体的な修正方法の説明

**修正コード例**:
```javascript
// 修正前
[問題のあるコード]

// 修正後
[推奨される修正コード]
```

**参考資料**:
関連する OWASP ガイドライン、CWE、CVE などへのリンク

## レビュー手順

1. 主要ファイルの Symbol Overview を取得
2. 認証関連ファイル（src/store/modules/auth.js、src/components/Login.vue）の詳細分析
3. API 通信関連ファイル（src/api/index.js、Axios 設定）の詳細分析
4. Vuex store モジュール（src/store/modules/*.js）でのデータ管理の確認
5. Vue コンポーネント（src/components/**/*.vue）での XSS 対策の確認
6. Vue Router（src/router/index.js）でのナビゲーションガードの確認
7. package.json の依存関係の脆弱性チェック
8. 環境設定ファイル（config/*.env.js）での機密情報の確認
9. 全体的なアーキテクチャとセキュリティ設計の評価

## 最終レポート構成

1. **エグゼクティブサマリー**
   - 発見された脆弱性の総数（重要度別）
   - 最も重大な問題のハイライト
   - 全体的なセキュリティ評価

2. **詳細な脆弱性リスト**（重要度順）
   - Critical レベルの脆弱性
   - High レベルの脆弱性
   - Medium レベルの脆弱性
   - Low レベルの脆弱性

3. **ベストプラクティスの推奨事項**
   - セキュリティヘッダーの追加
   - 入力検証ライブラリの導入
   - セキュリティ監視とロギングの強化
   - 定期的なセキュリティレビューの実施

4. **アクションプラン**
   - 優先順位付けされた修正ロードマップ
   - 短期的な対応（1週間以内）: Critical および High レベルの脆弱性
   - 中期的な対応（1ヶ月以内）: Medium レベルの脆弱性とアーキテクチャ改善
   - 長期的な対応（3ヶ月以内）: Low レベルの脆弱性とベストプラクティスの適用

## 重要な注意事項

- すべてのコードファイル（src/components/, src/store/, src/api/, config/）を対象にする
- ハードコードされた API エンドポイント、認証情報、API キーを特に注意深くチェック
- 古いパッケージバージョンによる既知の脆弱性を確認（Vue.js 2.5.16、Axios 0.24.0 など）
- プロジェクト固有の実装（JWT トークン管理、位置情報取得、ファイルアップロードなど）を重点的に分析
- localStorage や sessionStorage への機密情報保存をチェック
- v-html ディレクティブの使用箇所を重点的にチェック
- 修正コード例は、既存のコーディングスタイル（インデント、命名規則など）に従う
