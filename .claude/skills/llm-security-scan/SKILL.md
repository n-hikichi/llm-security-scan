# JavaScript Security Scanner

JavaScript/Node.js プロジェクトの包括的なセキュリティレビューを自動化するスキル。
プロジェクトの種類 (クライアントサイド/サーバーサイド) を自動検出し、
適切なセキュリティチェックリストに基づいてレビューを実施する。

## 主な機能

1. **自動プロジェクト検出**: package.json とディレクトリ構造からプロジェクトタイプを判定
2. **包括的なセキュリティチェック**: 認証、入力検証、XSS、SQL インジェクション、依存関係の脆弱性を網羅
3. **詳細なレポート生成**: 重要度別の脆弱性リスト、修正提案、コード例を含むレポート
4. **MCP Serena 統合**: Symbol Overview によるコード探索の効率化

## 使用方法

### 基本的なワークフロー

1. **プロジェクトディレクトリに移動**
   ```bash
   cd /path/to/your/project
   ```

2. **Claude Code CLI を起動**
   ```bash
   claude
   ```

3. **セキュリティスキャンを実行**
   
   スキルが自動的にプロジェクトを分析し、適切なセキュリティレビューを開始します:
   
   ```
   > このプロジェクトのセキュリティスキャンを実施してください
   ```

### プロジェクトタイプの指定

自動検出がうまくいかない場合は、手動でプロジェクトタイプを指定できます:

```
> クライアントサイドのセキュリティレビューを実施してください
```

または

```
> サーバーサイドのセキュリティレビューを実施してください
```

### MCP Serena との統合

より効率的なスキャンのために、MCP Serena を使用することを推奨します:

1. **MCP Serena のセットアップ**
   
   詳細は `references/mcp-serena-integration.md` を参照してください。
   
   > **前提**: スキャン対象 (SUT) ディレクトリで実行すること。`--project $(pwd)` で cwd を対象として登録します。

   ```bash
   claude mcp add serena -- $HOME/.local/bin/uvx --from git+https://github.com/oraios/serena serena start-mcp-server --context ide-assistant --project $(pwd)
   ```

2. **Onboarding の実行**
   
   ```
   > mcp serena 向けの onboarding 処理を行ってください
   ```

3. **セキュリティスキャンの実行**
   
   ```
   > セキュリティスキャンを実施してください
   ```

## セキュリティレビューの対象領域

### クライアントサイド (Vue.js, React など)

- **認証・認可**: トークン管理、ストレージのセキュリティ、自動ログアウト
- **XSS 対策**: ユーザー入力のエスケープ、v-html の使用、DOM 操作
- **API 通信**: HTTPS の強制、認証ヘッダー、エラーハンドリング
- **依存関係**: 既知の脆弱性、古いパッケージバージョン
- **データ保護**: localStorage/sessionStorage の使用、機密情報の露出
- **ルーティング**: 認証チェック、リダイレクト処理、URL パラメータ

### サーバーサイド (Express.js, Node.js など)

- **認証・認可**: JWT 管理、パスワードハッシュ、セッション管理
- **データベース**: SQL インジェクション、接続セキュリティ、暗号化
- **入力検証**: サニタイゼーション、型チェック、ホワイトリスト検証
- **依存関係**: 既知の脆弱性、セキュリティパッチ
- **CORS とヘッダー**: セキュリティヘッダー、CORS 設定
- **エラーハンドリング**: 情報漏洩、スタックトレース、ログ管理
- **レート制限**: DoS 対策、API 制限

## スキャン実行の流れ

1. **プロジェクト検出**
   - `scripts/detect_project_type.py` がプロジェクトを分析
   - package.json の依存関係とディレクトリ構造を確認
   - クライアント/サーバー/両方/不明を判定

2. **セキュリティプロンプトの生成**
   - `scripts/generate_scan_prompt.py` が適切なプロンプトを生成
   - `references/cli-security-prompt.md` (クライアント) または
   - `references/sv-security-prompt.md` (サーバー) を使用

3. **コード分析の実施**
   - MCP Serena (利用可能な場合) を使用して効率的にコードを探索
   - 主要ファイルの Symbol Overview を取得
   - 認証、API、データベース関連のファイルを重点的に分析

4. **脆弱性の検出と報告**
   - 重要度 (Critical/High/Medium/Low) を付けて分類
   - 各脆弱性について:
     - 場所 (ファイルパス:行番号)
     - 説明と影響
     - 修正提案とコード例
     - 参考資料 (OWASP, CWE など)

5. **レポートの生成**
   - エグゼクティブサマリー
   - 詳細な脆弱性リスト (重要度順)
   - ベストプラクティスの推奨事項
   - アクションプラン (短期/中期/長期)

## ベストプラクティス

1. **定期的なスキャン**: コードの変更ごとにセキュリティレビューを実施
2. **CI/CD への組み込み**: 自動化されたセキュリティチェックを設定
3. **優先順位付け**: Critical と High の脆弱性を優先的に修正
4. **依存関係の更新**: 定期的にパッケージの脆弱性をチェックし、更新する
5. **セキュリティヘッダー**: Helmet などを使用して適切なヘッダーを設定

## トラブルシューティング

### プロジェクトタイプが検出されない

- package.json が存在するか確認
- 手動でプロジェクトタイプを指定してスキャンを実行

### MCP Serena が利用できない

- MCP Serena なしでもスキャンは実行可能
- セットアップ方法は `references/mcp-serena-integration.md` を参照

### スキャンに時間がかかる

- 大規模プロジェクトの場合、特定のディレクトリに絞ってスキャンを実行
- MCP Serena の onboarding を事前に完了させておく

## 出力形式

スキャン結果は以下の形式で報告されます:

### エグゼクティブサマリー
- 発見された脆弱性の総数 (重要度別)
- 最も重大な問題のハイライト
- 全体的なセキュリティ評価

### 詳細な脆弱性リスト
各脆弱性について:
- **重要度**: Critical / High / Medium / Low
- **場所**: ファイルパス:行番号
- **説明**: 脆弱性の詳細
- **影響**: 潜在的なリスク
- **修正提案**: 具体的な修正方法
- **修正コード例**: 修正前後のコード
- **参考資料**: OWASP, CWE などへのリンク

### アクションプラン
- 短期的な対応 (1週間以内): Critical/High
- 中期的な対応 (1ヶ月以内): Medium
- 長期的な対応 (3ヶ月以内): Low とベストプラクティス
